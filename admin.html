Step-by-step setup

  1. Create your Airtable base. Recommended tables: Leads, Bookings, Users, Applications. Fields can mirror the JSON shape (see "Payload format" below).
  2. In Make.com, create a new scenario. Add a Webhooks > Custom webhook trigger. Copy the URL it generates and paste it into the field above.
  3. Add a Router after the webhook. Route by the type field. Each route handles one record type (lead, booking, application, user).
  4. For each route, add an Airtable "Create a Record" module pointing at the matching table. Map JSON fields from record.*. For bookings the guest is nested at record.guest.*.
  5. Add email modules to specific routes. Examples: on the welcome-popup route, send the WELCOME15 voucher via Gmail or SendGrid. On the booking-request route, send a confirmation email to record.guest.email.
  6. Turn the scenario on. Submit a test booking or signup and verify the data lands in Airtable and the email goes out. The "Test ping" button above sends a small fake payload to verify connectivity.

Payload format

Every webhook fires with the following JSON shape:

{
  "type": "leads",        // or "bookings", "users", "applications"
  "source": "ipartment-website",
  "record": {
    "id": "IPT-MA12B3C-XYZ",
    "createdAt": "2026-05-22T11:00:00.000Z",
    "source": "index.html",
    "email": "guest@example.com",
    "phone": "+84 123 456",
    "type": "welcome_popup_voucher",
    "voucher": "WELCOME15",
    "promoValue": "15% off first stay"
    // ... booking records also have: room, checkin, checkout, nights,
    //     total, guest:{first,last,email,phone}, etc.
  }
}

Things to know

  • Data still saves locally too. The webhook is fire-and-forget. If Make is down or unreachable, the record is still kept in this admin dashboard.
  • CORS: Make webhooks accept POSTs without preflight, so no special setup is needed.
  • Webhook URL is per-browser. If you open admin.html on a different device, you need to paste the URL there too (or share access via the Make scenario itself).
  • Right now this CRM is a simulator. When the webhook is set, it becomes real, persistent, and ties into your Airtable + email automation.
